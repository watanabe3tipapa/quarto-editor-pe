## ファイル操作

### ドキュメントを読み込む

1. ヘッダーの「UPLOAD」ボタンをクリック
2. `.qmd`、`.md`、`.txt` ファイルを選択
3. エディタに内容が表示されます

### ドキュメントを保存する

1. ヘッダーの「SAVE」ボタンをクリック
2. `quarto-document.qmd` としてダウンロードされます

### HTMLとしてエクスポート

1. ヘッダーの「RENDER (DL)」ボタンをクリック
2. 確認ダイアログ:
   - **OK**: Quarto CLI使用 (要サーバー起動)
   - **キャンセル**: markedライブラリ使用
3. `quarto-document.html` としてダウンロードされます

### ドキュメントをクリアする

1. ヘッダーの「CLEAR」ボタンをクリック
2. 確認ダイアログが表示されるので「OK」をクリック
3. エディタの内容がすべて削除されます

## 編集とプレビュー

### Editモード

- 「Edit」ボタンをクリック
- Markdownドキュメントを編集できます
- シンタックスハイライト、コード補完、自動インデントが有効です

### Previewモード

- 「Preview」ボタンをクリック
- 整形されたドキュメントのプレビューが表示されます

## コードブロックの実行

### 基本的な流れ

1. エディタ内のコードブロックをクリックして選択
2. 右側にパネルが表示される
3. 「▶ 実行」ボタンをクリック
4. ステータス表示が変化

### ステータス表示

| 表示 | 意味 |
|------|------|
| ○ | 待機中 |
| ● | 処理中 |
| ✓ | 成功 |
| ✗ | エラー |

### 結果の確認

**成功時:**
- 緑色の枠で結果が表示されます

**エラー時:**
- 赤色の枠でエラー情報が表示されます
- エラーの種類に応じたヒントが表示されます

### リセット

「× リセット」ボタンをクリックすると、実行結果をクリアし、待機中の状態に戻ります。

### 対応言語

| 言語 | 実行方法 |
|------|----------|
| JavaScript (js) | ブラウザ上で実行 |
| Python (python, py) | Pyodide (WebAssembly) で実行 |

### 閉じる

「✕ 閉じる」ボタンで選択を解除し、右パネルを非表示にします。

## AIアシスタント

AIを使ってコードの解説や補完ができます。

### を開く

画面左下の「🤖 AI」ボタンをクリックします。

### コードを選択する

1. エディタのコードブロックをクリックします
2. 選択したコードがAIアシスタントに渡されます

### AI機能の使い方

| 機能 | 説明 |
|------|------|
| 📖 Explain | コードの意味を日本語で説明します |
| ✨ Complete | コードの続きを提案します |
| 🔄 Refactor | コードをリファクタリングします |

### チャットで質問する

テキスト入力欄に質問を入力して送信できます。

### 接続状態

AIアシスタント右上のインジケーターで確認できます:

| 色 | 状態 |
|----|------|
| 緑 | Ollamaに接続済み |
| 赤 | 接続できません |
| 黄 | 確認中 |

## Ollamaについて

AIアシスタントはOllamaというローカルAIサーバーを使用します。

### 事前準備

```bash
# Ollamaをインストール（macOS）
brew install ollama

# AIモデルをダウンロード
ollama pull llama3.2

# Ollamaを起動
ollama serve
```

### Ollamaが起動していない場合

AIアシスタントは「未接続」（赤色）の状態になり、使えません。

## Quarto CLIサーバー (オプション)

Quarto CLIを使用してHTMLを出力する場合:

```bash
# APIサーバー起動 (別のターミナル)
npm run server

# ブラウザで「RENDER (DL)」→ OK
```

## トラブルシューティング

### エディタが表示されない

- `npm run dev` で開発サーバーを起動しているか確認
- ブラウザで `http://localhost:5173` にアクセス
- ブラウザのコンソール（F12）にエラーがないか確認

### AIアシスタントに接続できない

1. Ollamaが起動しているか確認: `ollama serve`
2. モデルがダウンロード済みか確認: `ollama list`
3. 必要に応じてモデルをダウンロード: `ollama pull llama3.2`

### ファイルがアップロードできない

- 対応形式を確認: `.qmd`, `.md`, `.txt`

### Quarto CLIが使えない

- `npm run server` でAPIサーバーを起動
- または「RENDER (DL)」でキャンセルしてmarkedを使用
